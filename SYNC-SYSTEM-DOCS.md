## Descripci√≥n General

Sistema ETL de 3 fases para sincronizaci√≥n bidireccional entre SQL Server (servidor del cliente) y PostgreSQL (backend centralizado).

### Caracter√≠sticas Principales

- **Ejecutable standalone**: Un solo .exe en el cliente, sin dependencias

- **Bidireccional**: Upload (cliente ‚Üí backend) y Download (backend ‚Üí cliente)
- **Incremental o Full**: Configurable por tabla
- **Seguro**: Configuraci√≥n encriptada AES-256-GCM

## Mantenimiento

### Ver logs de sincronizaci√≥n

```sql
-- √öltimas 50 sincronizaciones
SELECT
  tabla,
  direccion,
  estado,
  "registrosAfectados",
  "duracionMs",
  "fechaInicio"
FROM sync_logs
WHERE "tenantId" = 'acme-001'
ORDER BY "createdAt" DESC
LIMIT 50;

-- Errores recientes
SELECT
  tabla,
  fase,
  mensaje,
  "errorDetalle",
  "fechaInicio"
FROM sync_logs
WHERE "tenantId" = 'acme-001'
  AND estado = 'error'
  AND "createdAt" > NOW() - INTERVAL '7 days'
ORDER BY "createdAt" DESC;

-- Estad√≠sticas por tabla
SELECT
  tabla,
  direccion,
  COUNT(*) as total_ejecuciones,
  SUM(CASE WHEN estado = 'exitoso' THEN 1 ELSE 0 END) as exitosas,
  SUM(CASE WHEN estado = 'error' THEN 1 ELSE 0 END) as errores,
  AVG("duracionMs") as duracion_promedio_ms,
  SUM("registrosAfectados") as total_registros
FROM sync_logs
WHERE "tenantId" = 'acme-001'
  AND "createdAt" > NOW() - INTERVAL '30 days'
GROUP BY tabla, direccion
ORDER BY tabla, direccion;
```
