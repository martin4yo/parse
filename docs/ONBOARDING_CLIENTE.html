OPCIÓN 2: Conector API Bidireccional (PULL/PUSH desde ERP) (28h)

# Tarea Horas Descripción
A2.1 Análisis de API del ERP cliente 4h Documentación, endpoints, autenticación
A2.2 Configurar conector PULL (importar datos desde ERP) 8h Órdenes de compra, maestros, etc.
A2.3 Configurar conector PUSH (exportar documentos procesados) 8h Envío automático al ERP
A2.4 Mapeo de campos (Parse ↔ ERP) 4h Wizard de configuración visual
A2.5 Testing y validación de sincronización 4h Pruebas bidireccionales

Consideraciones: Requiere API REST del ERP. Soporta OAuth 2.0, API Key, Basic Auth. Incluye sistema de staging para validación manual antes de importar.

OPCIÓN 3: Conexión LEGACY (Carpeta Local + Sistemas Legacy) (22h)

# Tarea Horas Descripción
A3.1 Configurar servicio de monitoreo de carpeta local 4h File watcher con detección de nuevos archivos
A3.2 Implementar procesamiento automático desde carpeta 3h Upload automático a Parse al detectar archivo
A3.3 Configurar carpetas de salida (procesados/errores) 2h Organización automática post-procesamiento
A3.4 Configurar reglas de filtrado por nombre de archivo 2h Patrones regex para clasificación
A3.5 Implementar sistema de logs y notificaciones 3h Registro detallado de operaciones
A3.6 Crear servicio Windows/Linux para ejecución permanente 4h Daemon/Service con reinicio automático
A3.7 Testing completo con volumen real de documentos 2h Pruebas de stress y validación
A3.8 Documentación técnica y manual de configuración 2h Guía de instalación y troubleshooting

Consideraciones: Ideal para clientes con sistemas legacy sin APIs o que reciben documentos por FTP/carpetas compartidas. Requiere acceso a servidor/PC cliente para instalación del servicio. Soporta múltiples carpetas monitoreadas y procesamiento batch.

Supuestos y Exclusiones

Supuestos

  1. Parse es una aplicación web completamente gestionada por Axioma
  2. El cliente proveerá usuarios para testing en tiempo y forma
  3. La capacitación del paquete base es remota vía videollamada
  4. El cliente tiene muestras de documentos representativas para entrenamiento
  5. Axioma gestiona toda la infraestructura de IA (API Keys, modelos, costos)

Exclusiones (NO incluido en el paquete base)

  • Integración con APIs externas/ERPs (se cotiza en Opción 2)
  • Desarrollo de funcionalidades custom no contempladas
  • Soporte extendido post-estabilización (se cotiza en Fase 7)
  • Procesamiento de emails (se cotiza en Opción 3)
  • API pública (se cotiza en Opción 1)

Riesgos y Mitigaciones

Riesgo Probabilidad Impacto Mitigación
Documentos con formato no estándar (calidad muy baja) Media Alto Testing exhaustivo en Fase 3, uso de Sharp para optimización, múltiples motores de IA
Precisión de IA no alcanza EL NIVEL esperado Baja Alto Ajuste de prompts, uso de pattern learning, combinación de múltiples modelos
Plan de cuentas del cliente está desactualizado Media Medio Validación en Fase 1, actualización antes de Fase 4
Resistencia al cambio de usuarios Baja Medio Capacitación gradual, piloto con usuarios clave, soporte extendido
